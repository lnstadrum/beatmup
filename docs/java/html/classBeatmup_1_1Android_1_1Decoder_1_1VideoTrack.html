void Beatmup.Android.Decoder.VideoTrack.setFrameAvailableCallback ( FrameAvailableCallback  callback)
inline

Sets frame available callback.

In processing mode (see Decoder constructor) the decoded frame texture given by getFrame() MUST BE PROCESSED within this callback, e.g. be rendered within a Scene. Namely updateTexImage() needs to be called on its respective SurfaceTexture, otherwise the decoder stalls in a deadlock.
